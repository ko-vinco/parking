# 🚗 검단 이웃 공유 주차 서비스 (Geomdan Parking Share)

검단 신도시 이웃 간의 편리한 주차 공유를 위한 React + TypeScript 기반의 일체형(SPA) 웹 애플리케이션입니다.  
동작에 필요한 모든 데이터는 브라우저의 **`localStorage`**를 통해 안전하게 유지되므로, **별도의 유료 데이터베이스(API 키)나 서버 세팅 없이 100% 무료로 배포하고 구동할 수 있습니다.**

---

## 🌟 주요 특징
- **100% 프론트엔드 단독 구동**: 서버나 DB가 필요 없어 유지보수 비용이 전혀 들지 않습니다.
- **로컬 스토리지 데이터 유지**: 주차장 등록, 예약 신청, 예약 취소 내역이 사용자 브라우저에 안전하게 저장됩니다.
- **반응형 웹 UI (Tailwind CSS)**: 데스크톱부터 모바일 화면까지 완벽하게 호환됩니다.
- **동네별 직관적 지도 매핑**: 지도 상의 핀을 클릭해 아치형 지도 좌표로 실시간 확인이 가능합니다.

---

## 🛠️ 개발 및 실행 방법 (로컬 PC에서 구동하기)

이 프로젝트를 개인 컴퓨터(로컬 환경)에서 구동하는 방법입니다.

### 1. 필수 준비물
- 컴퓨터에 **Node.js**가 설치되어 있어야 합니다. (버전 18 이상 권장)
  - [Node.js 공식 홈페이지](https://nodejs.org)에서 다운로드할 수 있습니다.

### 2. 프로젝트 다운로드 및 폴더 접속
1. AI Studio 우측 상단의 **설정(톱니바퀴) 아이콘** 또는 **수출(Export) 메뉴**를 클릭하여 **Download ZIP**으로 프로젝트를 전체 다운로드합니다.
2. 다운로드한 `.zip` 압축파일을 풀고, 터미널(CMD 또는 VS Code 터미널)을 열어 해당 폴더로 진입합니다.
   ```bash
   cd geomdan-parking
   ```

### 3. 의존성 패키지 설치
프로젝트 구동에 필요한 라이브러리(React, Lucide 아이콘, Tailwind 등)를 설치합니다.
```bash
npm install
```

### 4. 개발 서버 실행 (실시간 확인)
설치가 완료되면 개발용 실시간 미리보기 서버를 실행합니다.
```bash
npm run dev
```
- 터미널에 나타나는 주소(`http://localhost:3000` 등)를 웹 브라우저 주소창에 입력하면 앱이 정상 실행됩니다.

## 🚀 깃허브(GitHub)에 올리고 무료로 배포(상용화)하는 방법

이 앱은 서버가 필요 없으므로, **정적인 파일들을 무료로 올려서 전 세계 누구나 사용할 수 있도록 만들어주는 정적 호스팅 서비스**를 통해 즉시 배포할 수 있습니다!

### 방법 A. Vercel로 클릭 몇 번 만에 무료 배포하기 (가장 추천 ⭐)
정말 간편하고 빠른 무료 배포 서비스입니다. 깃허브에 코드를 올리기만 하면 자동으로 연동되어 단 1분 만에 끝납니다.

1. [Vercel 공식 홈페이지](https://vercel.com)에 접속하여 **GitHub 계정으로 로그인 (Sign up with GitHub)** 합니다.
2. 메인 대시보드에서 **[Add New...] -> [Project]** 버튼을 누릅니다.
3. 내 깃허브 저장소(Repository) 목록 중에서 이 프로젝트 저장소를 선택한 후 **Import**를 누릅니다.
4. Framework Preset이 **Vite** 또는 **Other**로 되어있는지 확인하고, 밑에 아무 설정도 건드리지 않은 채 **[Deploy]** 버튼만 클릭합니다.
5. 30초 내로 배포가 완료되며, 영구적으로 접속 가능한 `https://your-project-name.vercel.app` 형태의 고유 링크가 무료로 발급됩니다!

---

### 방법 B. GitHub Pages로 무료 배포하기 (전통적인 방법)
깃허브 자체에서 제공하는 사이트 호스팅 방식입니다.

1. 내 깃허브에 이 프로젝트 전체 파일을 푸시(Upload)하여 저장소를 만듭니다.
2. 터미널에서 다음 패키지를 설치합니다:
   ```bash
   npm install gh-pages --save-dev
   ```
3. `package.json` 파일의 상단에 `homepage` 주소를 추가합니다:
   ```json
   "homepage": "https://<깃허브_사용자이름>.github.io/<저장소_이름>",
   ```
4. `package.json` 의 `scripts` 항목에 배포용 명령어 두 개를 추가합니다:
   ```json
   "scripts": {
     "predeploy": "npm run build",
     "deploy": "gh-pages -d dist",
     ...
   }
   ```
5. 터미널에 아래 명령어를 입력하면 자동으로 빌드가 진행되고 `dist` 웹사이트 파일들이 깃허브의 전용 호스팅 브랜치로 배포됩니다:
   ```bash
   npm run deploy
   ```

## 📁 `dist` 폴더란 무엇인가요? 어떻게 준비하나요?

`dist` 폴더는 **"Distribution (배포)"**의 약자입니다.  
React와 TypeScript로 작성된 원본 코드들은 웹 브라우저가 직접 해석할 수 없으므로, 컴퓨터가 실시간으로 읽고 로딩할 수 있는 단독 `HTML, CSS, JS` 정적 파일들로 번들링(최적화 압축)하는 과정이 필요합니다.

### ⚙️ `dist` 폴더 수동으로 생성(빌드)하기
로컬 터미널에서 아래 명령어를 단 한 줄 실행하면 됩니다:
```bash
npm run build
```
- 실행 즉시 프로젝트 내부에 `/dist`라는 폴더가 새로 생겨납니다.
- 이 `/dist` 폴더 안에는 `index.html`, 그리고 최적화된 하나의 Javascript 파일과 CSS 파일들이 들어가 있습니다.
- 사설 웹서버나 일반 호스팅(예: Netlify, 가비아, 고도몰 등)에 웹사이트를 직접 업로드하고 싶을 때, 다른 복잡한 리액트 소스코드 필요 없이 **바로 이 `/dist` 폴더의 알맹이들만 웹서버 홈 경로에 올려주시면 구동이 끝납니다!**

---

### 💡 꿀팁: 깃허브(GitHub) 저장소에 올릴 때 가이드
깃허브 저장소에 코드를 업로드할 때는 용량이 매우 무거운 `node_modules` 폴더와 이미 빌드된 결과물인 `dist` 폴더는 올리지 않는 것이 원칙입니다. (이미 `.gitignore` 파일에 제외 정의되어 있습니다!)  
다른 사용자나 내 PC에서 다시 다운로드 받았을 때는 `npm install` 명령어 하나만 실행하면 필요한 라이브러리 파일들을 깃허브 대신 공식 패키지 저장소에서 초고속으로 자동 다운로드해 주기 때문입니다.
